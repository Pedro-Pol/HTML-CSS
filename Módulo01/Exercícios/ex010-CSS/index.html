CSS: Cascading Style Sheets


As folhas de estilo encadeadas, as CSS, são a parte do código nas quais são empregados seletores para realizar declarações, indicando como os elementos estéticos do documento devem ser apresentados. Existem três formas de escrever os códigos de CSS: inline, internal e external.

CSS  Inline

As declarações de estilo podem ser efetuadas dentro das tags de HTML, efetando-a apenas essa tag. Basta inserir o atributo global style no interior da tag, modificando seus seletores, conforme o necessário. Por exemplo, para escrever o termo style acima, empregou-se o seguinte código:

<span style="font: normal 13pt Consolas;">style</span>

CSS  Internal e External

Podem ser escritas declarações de estilo válidas para todo o documento HTML, declarando no documento (internal) ou em um documento de CSS em separado (external).

No primeiro caso, uma tag <style> é aberta dentro da tag <head>, normalmente após o título e antes do encerramento dessa secção. Todas as declarações inseridas nessa secção condensada, dedicada ao estilo e à apresentação do documento, valeráão para todo o documento. Se por exemplo, fossem declaradas alterações para a tag <span>, como no exemplo anterior, todas as tags <span> do documento seriam afetadas; não apenas uma.

Por exemplo, para editar a apresentação desse documento, foram trasncritas as seguintes declarações: 

<head>
        <meta charset="UTF-8">
        <meta http-equiv="X-UA-Compatible">
        <meta name="viewport"
        content="width=device-width, initial-scale=1.0">
        <title>CSS: Inline, Internal e External</title>
    
    <style>
        body {
            font:               normal 12pt Arial;
            text-align:         justify;
            margin:             50px;
            background-color:   rgb(206, 213, 219);
        }
        section {
            max-width:          550px;
            margin:             auto;
            padding:            50px;
            border-radius:      25px;
            background-color:   azure;
        }
        h1 {
            letter-spacing:     2px;
            text-align:         center;
        }
        h2 {
            margin-top:         35px;
            margin-bottom:      25px;
        }
    </style>
</head>
<body> ...


Em virtude de melhor organização, existe a possibilidade de criar um texto externo para depositar esses comandos gerais de CSS. Basta incluir, após a tag <title> a tag <link:css>, que criará uma tag <link> contendo os atributos href, hyperlink reference, e rel, relationship. Caso o documento não exista, o VSCode permitirá a criação automática de um desses arquivos. O código se parece do seguinte modo:

<link rel="stylesheet" href="style.css">

Quando um mesmo portal possuir muitas páginas e uma identidade visual comum, é mais eficiente criar um único arquivo externo que determinará todas as múltiplas páginas do que escrever em cada página os mesmo comando estéticos. Alterações eventuais também se tornariam desnecessariamente laboriosas com numerosas CSS locais para cada página. Além disso, é possível que um documento esteja ligado a mais de um arquivo de CSS externo.

Ordem de Preferência

É imortante ressaltar, como nota final, que as CSS transcrito na secção da tag <head>ou em um documento externo determinam as regras gerais de estilo e de apresentação do documento; enquanto que as CSS inseridas dentro das tags determinam regras específicas. Comandos específicos sempre tem preferência aos genéricos, inclusive quando uma tag é especificada por id ou class.
