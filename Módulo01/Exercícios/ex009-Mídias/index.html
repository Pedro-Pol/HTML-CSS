Mídias em HTML

Imagens Dinâmicas

Com a possibilidade de um mesmo portal ou programa ser acessado por diferentes dispositivos, com telas de diferentes tamanhos e processadores diferentes, é necessário considerar o tamanho das imagens em um documento HTML. As imagens dinâmicas são diferentes arquivos de imagens que se alternam de acordo com o tamanho atual da página, adaptando-se a diferentes gastos de banda do internauta.

Para que isso seja feito, é empregada a tag <picture>; que envolopa a tag <img>. Dentro do bloco em que é contiga a tag de imagem, adiciona-se quantas forem as variações da imagem original, inserindo a tag <source>. Dentro da tag, adiciona-se, pelo menos, dois atributos: media, que estabelece a dimenção que marca a mudança de um arquivo de imagem para o outro; e srcset, que faz referência a essa outra imagem substitutiva.

Por exemplo:


O código empregado:


<picture>
    <source media="(max-width: 850px)" srcset=
    "Imagens/img-p.png">
    <source media="(max-width: 1300px)" srcset=
    "Imagens/img-m.png">
    <img src="Imagens/img-g.png" alt="Imagem grande">
</picture>

Áudios

Para inserção de arquivos de áudio no documento, basta criar uma tag <source>, envelopada em uma tag <audio>. Assim como a inserção de uma imagem dinâmica, o áudio pode possuir várias fontes, sources, internas, para que um formato de áudio alternativo seja carregado, caso o navegador não suporte o formato de áudio principal. Os três formatos de áudio que abrangem quase todos os navegadores são o .mp3, .ogg e .wav. É preferível disponibiizá-los nesta ordem, preterindo os arquivos .wav, porque são os mais pesados entre os formato mencionados.

Os atributos da tag <audio> disponibilizam uma série de opções. O atributo controls mostra au internauta os botões de opções do arquivo de áudio reproduzido; o atributo autoplay inicia a reprodução do arquivo assim que a página é aberta; o atributo loop determina a repetição indefinida do áudio. Importante é o atributo preload, que determina o carregamento do arquivo de áudio, possuindo os seguintes valores:

  • auto  determinando o carregamento de todo o áudio, juntamente com a página. Representa o acréscimo perigoso ao gasto de banda do usuário, tornando o documento mais pesado.
  • metadata  determinando o carregamento de alguns dados informativos do aquivo de áudio, sem carregar sua substância. Esse é o valor recomendado, genericamente.
  • none  determina nenhum carregamento até o internauta pressionar o botão de play.

Por exemplo:


O código empregado: 


<audio preload="metadata" controls>
    <source src="audios/euamovoce.mp3" type="audio/mpeg">
    <source src="audios/euamovoce.ogg" type="audio/ogg">
    <source src="audios/euamovoce.wav" type="audio/wav">

    <p>Seu navegador não suporta os formatos de áudio 
    disponíveis (.mp3; .wav; e .ogg). Faça download do
    áudio <a href="audios/euamovoce.mp3" download=
    "euamovoce.mp3" type="audio/mpeg">aqui</a></p>
</audio>

Vídeos

Arquivo de Vídeo de Servidor Interno

O mesmo método aplicado aos arquivos de áudio inseridos aplica-se ao arquivos inseridos de vídeo. Uma série de tags <source>, indicando formatos alternativos de arquivos armazenados no dispositivo, é envelopada em uma tag <video>. Os atributos são os mesmos daqueles da tag <audio>, com o acréscimo de dois diferentes: height e width; altura e largura, respectivamente. Esses atributos determinam em seus valores a altura e a largura do quadro em que o vídeo será reproduzido. Além disso, o atributo poster possibilita inserir uma imagem de capa ao vídeo, indicando um documento como seu valor.

Os formatos de vídeo suportados são .mp4; .m4v; .webm e .ogv, dando preferência a ordem de arquivo mais leve. IMPORTANTE: os arquivos de vídeo reproduzidos em uma página consomem uma quantidade altíssima de banda, refletindo nos custos de hospedagem do portal. Para exemplificar por contraste: o vídeo exemplifcado tem, aproximadamente, 1 MB de espaço; consumindo, em uma situação hipotética de 500 acessos diários, 500 MB. Em contraste, todo este documento HTML pesa apenas 10 KB.

Por exemplo:


O código utilizado: 


<video preload="metadata" poster="Imagens/lars-thumb.PNG" 
    height="300px" width="450px" controls>
    <source src="video/comonotransito.mp4" type="video/mp4">
    <source src="video/comonotransito.m4v" type="video/mp4">
    <source src="video/comonotransito.webm" type="video/webm">
    <source src="video/comonotransito.ogv" type="video/ogv">
    
    <p>O seu navegador não possui compatibilidade com nenhum
    dos formatos de reprodução de vídeo disponíveis.</p>
</video>

Arquivos de Vídeo de Servidor Externo

Para solucionar o problema do alto gasto de banda inserindo um arquivo de vídeo do próprio servidor é referenciar uma fonte externa. Em outros termos, um portal que ofereça serviços de armazenamento de vídeo (como o Youtube ou o Vimeo), ao qual será arremetido o consumo de banda da reprodução do vídeo.

Para realizar a inserção destes vídeos basta clicar na opção compartilhar, slecionando a incorporação, que gera um código HTML para copiar e colar, no código-fonte, a tag <iframe>.

Considere os dois vídeos, respectivamente, do Youtube e do Vimeo, por exemplo:





Os códigos utilizados, respectivamente:

DuKNk93q-Uw?start=58" title="YouTube video player" frameborder="0" 
allow="accelerometer; autoplay; clipboard-write; encrypted-media; 
gyroscope; picture-in-picture; web-share" allowfullscreen></iframe>

portrait=0" width="450" height="283" frameborder="0" allow="autoplay;
fullscreen; picture-in-picture" allowfullscreen></iframe>
